## 2024-05-18 - Avoid N+1 Queries in Next.js Report API Endpoints
**Learning:** Next.js API endpoints generating aggregated reports (like Infaq and Tabungan) can suffer severe performance degradation from N+1 query patterns, specifically when looping with `Promise.all` or `for...of` loops over individual records to fetch related data (like payments or savings). Attempting to read string fields representing numbers dynamically from PostgreSQL using string concatenation causes hidden arithmetic bugs.
**Action:** Always replace iterative database queries in Next.js API endpoints with batched `inArray` queries combined with in-memory `Map` aggregations. Cast numeric values retrieved from DB to `Number()` before performing arithmetic inside Map reduction loops. Use `array.length > 0` checks before calling `inArray` to avoid empty array syntax errors in PostgreSQL.
