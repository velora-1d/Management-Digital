## 2025-02-12 - Drizzle ORM Upsert Limitations on Non-Unique Columns
**Learning:** The `schoolSettings` table in `src/db/schema.ts` lacks a unique constraint or index on the `key` column. As a result, Drizzle ORM's `.onConflictDoUpdate({ target: schoolSettings.key, ... })` cannot be used to elegantly upsert settings in a single database query. Attempting to use upsert features on non-unique columns results in database errors, requiring manual separation of `update` and `insert` statements within a transaction.
**Action:** When attempting to use upserts (`onConflictDoUpdate`) in Drizzle ORM, always verify the schema first to ensure the target column has a `unique()` or `uniqueIndex()` constraint. If it does not, fall back to batch fetching existing records and manually separating inserts and updates.
