## 2024-05-18 - SQL Injection via Database Restore Endpoint
**Vulnerability:** The application's `POST /api/settings/restore` endpoint read uploaded `.sql` backup files, split them by lines starting with `INSERT INTO`, and passed the resulting strings directly into Drizzle ORM's `sql.raw()` to execute them on the database. An attacker with a compromised superadmin account or via CSRF could upload a maliciously crafted `.sql` file to execute arbitrary SQL commands (e.g. `DROP TABLE`, `UPDATE users SET role = 'superadmin'`, etc.) and gain full control over the database.
**Learning:** Using `sql.raw()` with any unsanitized user input, including uploaded files like `.sql` dumps, bypasses all parameterized query protections and is extremely dangerous.
**Prevention:** Never parse and execute raw `.sql` text provided by users. Database backups and restores should either be handled by direct database administration tools (like `psql` or `pg_restore`) securely outside the application layer, or if implemented within the app, should use serialized formats like JSON where data is strictly typed and inserted via parameterized ORM queries.
