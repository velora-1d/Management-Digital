## 2024-05-24 - Stored XSS in SweetAlert2 Modals
**Vulnerability:** Found direct string interpolation of user-supplied data (e.g., `cat.name`, `cat.description`) within the `html` configuration option of `Swal.fire` modals in the frontend code. This allows an attacker who can input malicious data into these fields to execute arbitrary JavaScript when the modal is opened by an administrator or other user (Stored XSS).
**Learning:** SweetAlert2's `html` property evaluates strings as raw HTML. When populating forms inside SweetAlert modals with existing database records, developers often use template literals directly in the HTML string, unknowingly introducing XSS.
**Prevention:** Never use template literals (`${value}`) to inject untrusted database content directly into the `html` string of `Swal.fire`. Instead, structure the raw HTML with standard input elements, and use the `didOpen` lifecycle hook to safely set the values using the DOM API (e.g., `(document.getElementById('id') as HTMLInputElement).value = item.name`).
