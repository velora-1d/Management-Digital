## 2023-10-27 - Auth Bypass and Weak PRNG in reset-password API
**Vulnerability:** The `reset-password` API route was completely missing authentication/authorization checks because Next.js middleware only decodes the JWT payload without signature verification (for Edge runtime efficiency). This allowed attackers to craft forged JWTs and reset anyone's password. Additionally, `Math.random()` was used for generating new passwords, which is cryptographically weak.
**Learning:** In this architecture, Next.js middleware `decodeJwtPayload` does not perform signature verification. API routes MUST invoke `requireAuth()` and `requireRole()` to ensure true signature verification via `getAuthUser()`. Furthermore, `Math.random()` should never be used for password generation.
**Prevention:** Always invoke `requireAuth()` at the top of protected API routes, rather than relying solely on the middleware's initial routing check. Use `node:crypto`'s `randomInt` for secure random generation.
