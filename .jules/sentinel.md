## 2025-02-14 - Fix SQL Injection in Database Restore File Parsing
**Vulnerability:** The application blindly parsed and executed raw SQL queries from uploaded `.sql` dumps using `sql.raw()` in Drizzle ORM. If an uploaded file contained an `INSERT INTO` string, it would execute whatever followed without sanitization.
**Learning:** Using `sql.raw()` to execute contents derived from a file upload creates a critical SQL Injection risk. Simply checking if the file `.includes("INSERT INTO")` does not prevent multi-statement injection. String literals in SQL must be safely ignored/stripped during validation to accurately count statement-terminating semicolons.
**Prevention:** Always restrict dynamically parsed queries against an explicit whitelist of allowed tables (`ALLOWED_TABLES`). Enforce strict statement structure using Regex (`^INSERT INTO "[a-zA-Z0-9_]+"`) and safely strip valid string literals (`replace(/'(?:[^']|'')*'/g, "''")`) before counting semicolons to ensure exactly one semicolon exists at the end of the statement, neutralizing statement stacking injections.
