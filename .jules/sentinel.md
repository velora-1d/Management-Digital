## 2025-03-23 - Authentication Bypass in Profile Endpoints via Hardcoded User Fetch
**Vulnerability:** The profile and profile password endpoints (`/api/profile/me/route.ts` and `/api/profile/me/password/route.ts`) were hardcoded to fetch the very first active user (`getMe()`) from the database and perform profile updates and password changes on that user. This bypassed authentication, meaning any user, regardless of their legitimate session, could modify the first user's profile and password.
**Learning:** Hardcoding a generic user fetch (like a `getMe()` mock) bypasses true session management and results in severe IDOR/Authentication bypass vulnerabilities. It is critical to ensure that when taking actions "on behalf of the current user", the identity is explicitly derived from their validated session context.
**Prevention:** Always use established session management tools, such as the `requireAuth()` helper, which validates the JWT token and extracts the `userId`. The extracted `userId` must be used to specifically fetch or update the record belonging to the authenticated session owner.
