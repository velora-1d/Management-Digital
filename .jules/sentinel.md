## 2024-05-18 - XSS in Inventory Dashboard Page
**Vulnerability:** Found `dangerouslySetInnerHTML` being used in `src/app/(dashboard)/inventory/page.tsx` to render string-based HTML badges based on the item's condition. While the current inputs ("Baik", "Rusak Ringan", "Rusak Berat") were controlled by application logic, this pattern introduces a severe risk of Cross-Site Scripting (XSS) if the input logic changes to incorporate user data directly into the HTML string without proper sanitization.
**Learning:** Using `dangerouslySetInnerHTML` with dynamically constructed strings, even if currently safe, sets a dangerous precedent and violates React security best practices. React inherently protects against XSS when rendering strings as text content, but bypassing this mechanism opens the door to potential exploitation if the data source becomes untrusted.
**Prevention:** Avoid `dangerouslySetInnerHTML` whenever possible. Use standard React component logic (JSX) to conditionally render DOM elements and attributes. If dynamic HTML injection is absolutely necessary, strictly sanitize the input using a robust library like DOMPurify before rendering.
