## 2024-05-04 - Insecure Randomness and Authentication Bypass in Password Reset
**Vulnerability:** The `/api/users/[id]/reset-password` endpoint generated passwords using `Math.random()`, which is cryptographically insecure and predictable. Furthermore, the endpoint was missing full JWT signature verification and authorization checks, relying solely on Next.js middleware which does not verify JWT signatures.
**Learning:** In Next.js applications where middleware performs only basic JWT decoding for routing, API routes must explicitly call `requireAuth()` to verify the token signature. Additionally, always use `node:crypto` (`randomInt`, `randomBytes`) for security-sensitive random generation.
**Prevention:** Use established cryptographic libraries (`node:crypto`) instead of `Math.random()` for passwords. Always enforce signature-based authentication (`requireAuth`) and strict role-based authorization (`requireRole`) at the API route level to ensure defense in depth.
