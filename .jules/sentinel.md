## 2024-05-18 - Disable In-App SQL Restore Feature
**Vulnerability:** The application had an endpoint (`src/app/api/settings/restore/route.ts`) that parsed uploaded `.sql` files and directly executed them using Drizzle's `sql.raw()`. This poses a critical SQL injection risk, as an attacker could potentially upload a malicious `.sql` file executing unauthorized commands against the database.
**Learning:** Using `sql.raw()` with unsanitized file inputs bypasses all ORM safeguards and introduces critical Remote Code Execution / SQL Injection vulnerabilities. Full database restore operations are complex and risky to implement securely within the application layer.
**Prevention:** Disable application-level raw SQL execution for full database restores. Full database restores should exclusively be performed using established, secure, dedicated database management tools (such as pgAdmin, `psql`) outside of the web application. Never pass unsanitized user-uploaded content to `sql.raw()`.
