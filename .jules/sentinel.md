## 2024-05-18 - Fix XSS in SweetAlert2 modals and React dangerouslySetInnerHTML
**Vulnerability:** The application was vulnerable to Cross-Site Scripting (XSS) due to two practices: 1) Using React's `dangerouslySetInnerHTML` to render badges. 2) Using template literal string interpolation with unsanitized database content inside SweetAlert2 (`Swal.fire`) `html` options.
**Learning:** Even though the database content originates from user input that might be validated, relying on string concatenation or `dangerouslySetInnerHTML` directly in the UI layer without proper sanitization opens a severe attack vector for XSS if malicious scripts are injected into the database. SweetAlert2 parses the `html` string as raw HTML, making string interpolation dangerous.
**Prevention:** To prevent this, avoid `dangerouslySetInnerHTML` where standard React JSX variables and elements can be used. When using SweetAlert2, render static HTML structure in the `html` string and populate dynamic or user-controlled values securely using DOM manipulation (e.g., `(document.getElementById('id')).value = ...`) within the `didOpen` lifecycle hook.
